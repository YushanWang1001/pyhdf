pysclint - A collection of python interfaces to major scientific libraries

Introduction

pysclint is a set of python packages aimed at providing high quality python interfaces  to major scientific libraires like netCDF, pyhdf, and MPI. The packages are written so as to stay as close as possible to the original libraries, trying to cover as much of the libraries API's as possible, with no intent to heavily redesign them or rewrite their documentation.  At the same time, pysclint  tries to wrap the API's in as a natural pythonic paradigm as possible, augmenting it with  an OOP approach whenever appropriate. A major goal was to make a user of the original API (usually in C) feel at home with the python version, while not requiring any previous experience with this API when first learning it through the python language.

The following interfaces are currently provided.

  • pycdf - Python interface to the Unidata netcdf library.
  • pyhdf - Python interface to the  NCSA hdf4 library.
An interface tothe MPI 1.2 standard will be provided in a coming future.

pycdf

pycdf is a python interface to the Unidata netCDF library.  It provides an almost complete coverage of the netCDF C API, wrapping it  inside easy to use classes. netCDF arrays are handled using Numeric Python array objects.

netCDF versions up to 3.5.0 have been successfully tested with pycdf, under Python v2.2 and above .

pyhdf

pyhdf is a python interface to the NCSA HDF4 library. Among the numerous components offered by HDF4, the following are currently supported by pyhdf : SD (Scientific Dataset), VS (Vdata), V (Vgroup) and HDF (common declarations). SD arrays are handled using Numeric Python array objects.

HDF versions up to 4.2r1 have been successfully tested with pyhdf, under Python v2.2 and above. Note that there exists a version 5 of HDF, which is totally incompatible with version 4. pyhdf does not work with HDF5.
